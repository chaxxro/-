# 核心概念

## LLM

LLM 就是一个读过海量资料、会接话、会写代码的超级助理大脑，它本身不懂对错、不理解业务，只是在做在当前上下文中，最有可能出现的下一个词是什么

LLM 只是一个大脑，要干活还得配手和规则

常见的 LLM 有：GPT、Claude、Gemini 等

## Agent

因为 LLM 只是一个大脑，所以需要 agent 替它完成其他操作

用户直接跟 agent 交互，由 agent 与 LLM 交互并且处理 LLM 无法操作的工具调用，这样可以减少用户直接跟 LLM 交互的次数

## Function calling

LLM 与 agent 之间关于工具调用所约定的对话格式就是 function calling，类似于前后端约定的接口格式一样

## MCP

搜索、文件读取、脚本执行、pdf 转换等工具当前是集成在 agent 里，没有与 agent 解耦。如果可以把这些能力从 Agent 中解耦，就可以将决策逻辑和执行能力分离，实现能力可插拔、风险可隔离、权限可控

模型上下文协议 MCP 是给大模型统一接外部能力的插线板标准

在 MCP 之前每个 AI 工具都要自己接 Git、数据库、内部系统，接法不统一，重复造轮子。

LLM 就像一个只会说话，不会动手的天才，MCP 可以提供各种工具的程序集，中间的 agent 是一个传话筒，把 LLM 的话转换成调用工具的代码，然后 agent 再把工具调用的结果再原封不动的传话给 LLM，同时 agent 还要把结果返回给用户

## Skills

Skills 其实就是一个 prompt 预加载器，skill.md 是一个统一的说明文件，把整体的流程描述清楚

## SubAgent

对于一个复杂的任务，可能 agent 的上下文会变得非常大

对于一些独立的子任务，可以单独在这个子 agent 中完成。本质上是做了上下文隔离，子 agent 产生的上下文不会保留在主agent中

## Rules

Rules 是约束大模型别乱来、别自作聪明的行为规范，把人的经验，变成 AI 必须遵守的约束

LLM 的一个天性是太积极、太自信、太爱发挥，所以必须立规矩

Rules 就是在告诉 LLM 你是谁、你能干什么、你不能干什么、遇到不确定时怎么办

## 总结

如今的大多数技术，归根结底仍是围绕大模型展开，其核心依赖的提供的提示词

诸如 Search、RAG、Skill 等方案，本质上都是以自动化的方式向大模型填充上下文信息，从而减少与大模型的交互次数，提高效率
