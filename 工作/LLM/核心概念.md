# 核心概念

## LLM

LLM 就是一个读过海量资料、会接话、会写代码的超级助理大脑，它本身不懂对错、不理解业务，只是在做在当前上下文中，最有可能出现的下一个词是什么

LLM 只是一个大脑，要干活还得配手和规则

常见的 LLM 有：GPT、Claude、Gemini 等

## Skills

Skills 是让大模型真的能干活的具体能力按钮，是 LLM 的工具和技能

没有 Skills 的 LLM 只能聊天，只能建议，不能落地

## MCP

MCP 是给大模型统一接外部能力的插线板标准

在 MCP 之前每个 AI 工具都要自己接 Git、数据库、内部系统，接法不统一，重复造轮子

## Rules

Rules 是约束大模型别乱来、别自作聪明的行为规范，把人的经验，变成 AI 必须遵守的约束

LLM 的一个天性是太积极、太自信、太爱发挥，所以必须立规矩

Rules 就是在告诉 LLM 你是谁、你能干什么、你不能干什么、遇到不确定时怎么办

## Vibe-coding

Vibe-coding = 我大概说个意思，你自由发挥写代码

典型对话是：帮我写个登录功能、大概差不多就行...然后 AI 写一大堆，能跑但不一定对。适合 Demo、原型验证，不适合复杂业务、长期维护

## Spec-coding

Spec-coding = 我先写清楚规格，你严格按说明实现

先给 AI 一个明确说明书，要求精确控制每一个细节, 然后 AI 才开始写代码

- rules 是长期规则（全局）
- spec 是一次任务的临时规格
